# 翻译APP MVP 1.0 产品需求文档（PRD）

## 一、文档概述

### 1.1 文档目的

本文档为翻译类APP MVP 1.0版本正式产品需求文件，完整明确产品核心功能定义、页面布局规范、全流程交互逻辑、细节约束规则，作为产品研发、UI设计、功能测试、后续版本迭代的唯一核心依据，同步固化全部已确认需求，避免需求信息偏差与遗漏。

### 1.2 版本约束

- 产品版本：MVP 1.0 最小可行版本

- 核心设计原则：极简工具化定位、贴合用户操作直觉、满足监管合规要求，仅落地核心刚需功能，非核心功能延后迭代。

## 二、产品定位

本产品定位为轻量化、多场景专业翻译工具，核心服务于在校学生、医疗/法律等行业专业人士及日常翻译需求用户，支持文本、URL、文档、图片、课堂实时翻译五大核心场景，兼顾翻译效率与长效留存，区别于纯词典类产品，强化专业翻译工具属性，实现「高效翻译+轻量云盘留存」的核心价值。

## 三、核心目标用户

1. **学生用户**：高频处理外文学习资料、学术文献，对文本/文档翻译有刚需，需长效留存查阅；

2. **行业专业用户**：医疗、法律等领域从业者，需精准翻译专业文档，看重文件留存能力；

3. **大众通用用户**：有日常文本、网页翻译需求，追求操作极简、高效直达。

## 四、全局通用设计规范

1. 视觉规范：极简工具类视觉风格，无冗余装饰元素，功能入口清晰直观，全产品交互逻辑统一；

2. 交互规范：核心操作一键直达，返回逻辑统一，无异常跳转；

3. 状态提示规范：所有加载、失败、操作成功提示统一文案，2秒自动消失，不干扰核心操作；

4. 权限规范：仅申请产品正常运行必需权限（存储、相机、麦克风、网络），无多余权限申请。

## 五、核心页面详细需求说明

### 5.1 【首页】（产品核心主页面）

#### 5.1.1 页面布局规范

页面采用自上而下垂直布局，层级顺序：顶部（中间360 AI翻译logo + 右侧【我的翻译库】入口） → 核心翻译输入框 → 输入框附属快捷操作 → 核心功能快捷入口 → 底部导航Tab栏；布局简洁规整，无冗余元素，功能分区清晰。其中360 AI翻译logo位于顶部中间位置，与右侧【我的翻译库】入口同层级，仅作为品牌标识展示；核心翻译输入框去掉原有边框，采用到边设计并添加浅灰色背景以明确输入范围，语言对切换模块移至输入区底部外部。

#### 5.1.2 各模块详情

1. **顶部功能入口**：顶部仅包含两个核心元素，同层级布局——中间为360 AI翻译logo，右上角设置【我的翻译库】入口；点击【我的翻译库】入口跳转至对应页面，承载历史记录与文件留存功能；小屏手机可将【我的翻译库】入口简化为图标（无文字），避免顶部布局拥挤。

2. **360 AI翻译icon**：位于【首页】顶部中间位置，与右上角【我的翻译库】入口同层级，独立展示，仅作为品牌标识，不触发任何交互；控制logo尺寸，优先展示“360 AI翻译”核心文字，避免过大占用顶部空间，适配各类屏幕尺寸。

3. **核心翻译输入框**：【首页】核心模块，去掉原有边框，采用谷歌式到边设计，添加浅灰色背景（极淡、不突兀），与页面白色背景轻微区分，明确输入范围，方便用户识别可点击区域；语言对切换功能（默认英文→中文）移出输入框，移至输入区底部外部，语言对模块单独添加明显背景色（与输入区浅灰背景区分），与输入区底部保留5-10px间距，整体与输入区左右对齐、居中排布；语言对展示：默认显示「英文→中文」，文字右侧配套切换箭头图标，点击弹出【语言选择列表】，支持自由切换源语言与目标语言，点击切换箭头可互换源语言与目标语言，功能与原有逻辑完全一致。

4. **输入交互：**结合高效输入交互逻辑，贴合调整后的输入区与语言对位置，优化输入全流程体验，具体如下：

    - 默认状态：核心翻译输入框显示内置置灰提示，键盘处于未唤起状态，键盘右下角「翻译」/「go」按钮默认置灰、不可点击；

    - 键盘唤起：用户点击核心翻译输入框任意区域（含浅灰色背景区域），立即唤起键盘；键盘弹出后，【首页】整体可视区域同步收窄，输入区同步上移，语言对模块（带独立背景色）随输入区上移，自然落在键盘顶部，避免被键盘遮挡，输入框附属快捷操作、核心功能快捷入口、底部导航Tab栏保持原有位置和样式不变；

    - 输入与按钮状态：用户在核心翻译输入框内手动输入或粘贴文本/URL时，同步触发实时翻译（贴合产品原有自动翻译逻辑），输入内容实时显示，置灰提示自动消失；同时，键盘右下角原本置灰的「翻译」/「go」按钮同步变为蓝色可点击状态，提示用户可触发跳转；

    - 跳转逻辑：用户点击键盘右下角蓝色可点击状态的「翻译」/「go」按钮后，根据输入内容跳转至对应结果页——输入文本（单词/短句/长文本）跳转至【文本翻译结果页】，输入URL（网页链接）跳转至【网页翻译结果页】；两个结果页均为独立页面，与【首页】存在明确跳转交互，跳转过程无额外干扰，符合全局交互规范。

5. **提示文案：**内置置灰提示「粘贴需要翻译的文本、URL，或手动输入内容」，不遮挡输入区域；当用户在输入框内输入对应的文字后，该置灰提示自动消失。

6. **输入框附属快捷操作**：【首页】核心翻译输入框内部右下角，布局两个功能入口（均在输入框内），分别为上传入口和语音输入入口，规整排布；其中上传入口为单个按钮，点击后从【首页】底部弹出弹层，供用户选择文档上传或图片上传；语音输入入口点击后，直接触发语音输入样式，全程操作闭环于【首页】当前页面，无页面跳转。

7. **核心功能快捷入口**：【首页】设置【文档翻译】、【图片翻译】、【课堂翻译】三个入口，横向平铺排布，点击直接跳转至对应功能独立页面；

8. **底部导航Tab栏**：【首页】底部仅设置【首页】、【我的】两个核心Tab，默认选中【首页】，点击【我的】跳转至【我的】页面。

### 5.2 【文本翻译结果页】

#### 5.2.1 页面核心定位

独立页面，仅承接【首页】文本输入（手动输入、粘贴文本）触发的翻译需求，聚焦文本类（单词 / 短句 / 长文本）翻译结果展示，交互贴合文本翻译场景，与【网页翻译结果页】明确区分。

核心目标：满足用户文本翻译、朗读、复制及连续翻译需求，整体遵循极简工具定位，无冗余操作与元素。

#### 5.2.2 展示与交互规范

1. **feature展示规范**：

所有文本翻译内容均以单个feature形式展示，核心包含2部分，无语言切换区域（语言对独立存在）：

- 核心内容区域：源语言文本（中英场景附加英、美音标）与目标语言译文上下对照排布，两者之间有明确样式区分；单个feature设置固定高度，文本过长时可在各自区域内上下滑动，不拉伸整体高度。

- 功能入口及交互：

    - 基础操作：源语言、目标语言旁各设1个「朗读」按钮，目标语言右下部设「复制」按钮；点击「复制」弹出2秒自动消失的toast提示「已复制到剪切板」，翻译完成不显示任何成功提示。

    - 朗读细节：无论文本长短、是否滑动，点击「朗读」按钮均完整朗读对应区域全部文本，与文本可见范围无关。

    - **朗读与暂停规范（核心约束）**：

        - 互斥规则：同一时间仅允许播放一段文本，不支持双语同时朗读；

        - 状态切换：点击「播放」按钮→切换为「暂停」按钮并开始朗读，再次点击「暂停」→停止朗读并恢复为「播放」按钮；

        - 跨模块切换：若某一语言正在朗读（按钮为暂停状态），直接点击另一语言的「播放」按钮，将立即终止当前朗读、恢复其播放按钮，同步启动新语言朗读并切换其按钮为暂停状态。

1. **独立语言对模块规范**：

语言对模块独立于所有feature，不依附任何一个feature（无论已完成翻译/空白），作为页面独立模块存在，核心规则如下：

- 语言继承：从【首页】继承源语言、目标语言选择，不默认显示「英文→中文」，完全同步用户在首页的选择。

- 视觉规范：沿用【首页】语言对风格，单独添加明显背景色，与页面其他区域（feature、空白区）明确区分，布局与页面左右对齐、居中排布。

- 功能及联动逻辑：

    - 基础功能：支持点击弹出【语言选择列表】切换语言，点击切换箭头互换源语言与目标语言，与首页交互逻辑一致；

    - 联动规则：若当前页面存在「空白feature」（未在源语言输入区输入任何内容的feature，无论数量多少），当用户点击独立语言对模块进行切换操作（切换语言、互换语言）时，所有空白feature将同步切换为当前最新的语言对（与独立语言对模块切换后的语言对完全一致）；已完成翻译的feature，语言对保持翻译时的状态，不跟随切换。

    - 交互效果：语言对切换时，独立语言对模块有轻微淡入淡出动画（贴合极简风格，不突兀）；同时，所有空白feature的语言标识区域同步出现短暂闪烁（1秒），明确告知用户“空白feature已同步切换语言对”，无额外冗余提示。

- 位置规则（仅与键盘状态关联，与feature无关）：

    - 键盘展现时（无论何种方式唤起，如点击空白feature、粘贴内容）：语言对模块固定展示在「键盘上方」，位置不随feature的数量、位置变化，避免被键盘遮挡；

    - 键盘回收时（无论何种方式回收，如滑动屏幕、点击空白处）：因页面无底部导航，语言对模块固定置于「页面最底部」，贴紧页面底部边缘。

1. **「下一个」（英文：the next）交互规则**：「下一个」入口仅与feature状态关联，与独立语言对模块无关，核心规则如下：

    - 入口位置：不设置在页面底部，仅在每个已完成翻译的feature模块底部展示（中文「下一个」、英文「the next」）。

    - 置灰规则：新增的空白feature（源语言/目标语言均为空），其底部「下一个」入口置灰、不可点击；仅当用户在空白feature的源语言输入区填写内容（无论是否完成翻译），入口才变为可点击状态。

    - 点击交互：点击可点击状态的「下一个」，立即新增1个空白feature（源语言、目标语言均为空），同时前一个已完成翻译的feature底部「下一个」入口自动消失；新增空白feature完成翻译后，其底部自动显示「下一个」入口（初始置灰，需输入内容后可点击），确保始终只有最新的feature底部有「下一个」入口。

    - 光标与键盘状态：

        - 若点击「下一个」时键盘处于弹出状态：光标自动停留在新feature的源语言输入区，键盘保持弹出；

        - 若点击「下一个」时键盘已回收：仅新增空白feature，不弹出键盘、不展示光标。

    - 翻译触发机制（与首页一致）：

        - 粘贴触发：在空白feature源语言区粘贴内容后，自动触发翻译，无需额外操作；

        - 输入触发：手动输入内容完成后，自动触发翻译；

        - 手动触发：点击键盘右下角「go」或「翻译」按钮，手动触发翻译；

        - 提示规则：翻译完成不显示成功提示，仅点击复制时弹出2秒toast提示。

2. **通用交互及页面特性**：

    - 基础交互：页面顶部设返回箭头，点击直接返回【首页】，无其他多余操作，贴合极简定位。

    - 核心操作保留：翻译完成后，仅保留朗读、复制、语言切换3个核心操作，无冗余信息展示。

    - 页面特性（一次性页面）：

        - 无feature删除入口；

        - 用户点击返回箭头回到【首页】后，当前页面所有feature模块、独立语言对模块自动消失；

        - 再次从首页输入文本跳转至本页面时，页面重新初始化，仅展示1个空白feature（底部「下一个」入口置灰）及独立语言对模块（语言对继承首页选择，初始位置随键盘状态变化），无任何历史feature记录。

3. **补充说明**：

    - 所有模块互不干扰：feature展示、独立语言对、「下一个」入口、通用交互各自独立，仅遵循自身逻辑，不交叉影响；

    - 联动逻辑约束：仅空白feature（未输入内容）跟随独立语言对切换，已完成翻译的feature语言对固定不变，避免用户误操作导致历史翻译内容混乱；

    - 交互效果极简：仅添加轻微动画和闪烁提示，不添加多余弹窗，贴合产品极简工具定位；



### 5.3 【网页翻译结果页】

#### 5.3.1 页面核心定位

独立沉浸式全屏页面，以 WebView 为核心载体，承接首页 URL 输入跳转场景。采用**原生网页完整保留 + 原位逐段嵌入式翻译**方案，不破坏网页排版、CSS 与原生交互；顶部适配灵动岛 / 安卓状态栏、支持下拉刷新动效；底部配备双状态「底部功能操作栏」+ 全局加载进度条；语种默认跟随系统配置，同时支持用户自定义修改源语言、目标语言；自带翻译 / 原文一键切换、划词翻译、网页图文全量翻译能力，全页面视觉 & 交互规则与首页、文本翻译结果页统一。

#### 5.3.2 页面基础布局

##### 一、顶部适配 & 下拉刷新交互

1. 机型基础适配

    - iOS 机型：顶部保留灵动岛适配区域，预留固定安全高度，不遮挡灵动岛、状态栏内容，常态下该区域无任何元素、纯干净留白；

    - 安卓机型：顶部适配系统状态栏高度，常态无功能按钮、无文案，仅做系统 UI 避让。

2. 标准下拉刷新逻辑用户在网页区域**按住屏幕向下拉动**，拉动距离达到临界阈值后，自动触发页面全局刷新；拉动未达阈值松手，无任何刷新动作，回弹还原。

3. 刷新状态视觉 & 动效细节

    - 刷新触发后：iOS：在**灵动岛正下方**独立区域，展示标准环形旋转 Loading，不侵入、不遮挡灵动岛本体；安卓：在顶部状态栏下方居中位置，展示同款环形旋转 Loading；

    - 进度联动：顶部环形 Loading 动画，与屏幕最底部常驻网页加载进度条**全程同步运转、同步结束**；

    - 加载完成收尾动效：网页资源、翻译任务全部加载完毕后，顶部环形 Loading 执行收缩、渐隐动画；iOS 专属：Loading 收缩收尾后，**平滑收纳回收至灵动岛内部**，完全隐藏无残留；安卓专属：Loading 垂直向上平滑收起、淡出消失，顶部回归纯净留白状态。

##### 二、中间核心 WebView 区域

1. 基础规则整体全屏 WebView 渲染，完整保留网页原生滑动、点击、缩放、懒加载、跳转等能力；**完全不篡改网页原有排版、布局、DOM、CSS 样式**，仅做译文嵌入式插入，杜绝乱版错位。

2. 原位逐段翻译交互

    - 拆分颗粒度：以标题、大段落、独立短句为单个翻译单元；

    - 翻译中状态：每个文本单元翻译请求发起后，**在原文下一行位置展示小型环形菊花 Loading**，简洁轻量化，不遮挡原文、不破坏排版；

    - 翻译完成：单段译文返回后，菊花 Loading 自动消失；

3. 划词翻译长按网页文本唤起划词选中态，弹出极简浮层，支持对照译文、朗读、复制；语种同步当前页面已选定的源 / 目标语言。

整体包含 3 个核心关联页面，流程闭环：【文档上传 + 语言选择页】→【文档上传加载页】→【文档翻译结果页】，三个页面均遵循 “顶部返回箭头 + 核心功能区 + 底部无冗余元素” 的布局，视觉风格（胶囊、颜色、动效）与首页、网页翻译页完全统一。

##### 一、【文档上传 + 语言选择页】（核心入口页）

1. 顶部区域：左侧设置返回箭头，点击直接返回【首页】，无其他冗余元素；顶部中间展示 “文档翻译” 标题，字体、尺寸贴合全局标题规范。

2. 核心功能区（同一页面整合上传 + 语言选择，不拆分独立页面）：

    - 上传区域：居中展示 “点击上传文档” 按钮（贴合全局按钮样式，蓝色主题），下方标注支持的文档格式及大小限制（清晰标注：pdf、ppt 等 11 种格式≤100M；txt、html 等 4 种格式≤1M），字体轻量化，不突兀。

1. 触发时机：用户在【文档上传 + 语言选择页】选定语言、点击上传按钮→跳转系统文件管理器→用户选定文档后，返回 APP 自动进入本页面。

2. 布局：全屏加载页面，无多余元素，居中展示 “正在上传” 文案 + 环形菊花 Loading（与网页翻译的 Loading 样式完全一致，轻量化），Loading 下方同步显示上传进度（百分比，贴合全局轻量加载规范）。

3. 流程衔接：上传完成后，自动弹出 2 秒轻量 Toast 提示「上传成功，正在翻译」，Toast 消失后，直接跳转至【文档翻译结果页】，无额外操作，流程闭环。

##### 三、【文档翻译结果页】（核心展示页）

1. 顶部区域：左侧设置返回箭头，点击返回【文档上传 + 语言选择页】；顶部无语言对模块（语言已在上传前选定，不重复展示），贴合你的要求。

2. 核心展示区（文档编辑器形态）：

    - 切换功能：核心展示区顶部设置「原文」「译文」两个 Tab，横向排布，默认选中「译文」，点击 Tab 可自由切换原文、译文展示，切换时无卡顿，贴合极简交互。

4. 底部区域：无固定操作栏，仅在导出按钮展开时显示，避免遮挡文档内容，贴合阅读场景需求。

    - 默认规则：【文档上传 + 语言选择页】语言对默认「自动识别文档源语言→系统目标语言」，贴合系统语言设置；

    - 手动切换：点击语言对箭头，弹出语言选择列表，可双向切换源语言、目标语言，切换后不影响已上传操作（仅影响后续翻译结果）。

2. 翻译异常：

    - 翻译失败（网络异常、文档内容异常）：弹出 2 秒 Toast 提示「翻译失败，请重试」，页面保留 “重新翻译” 按钮（轻量化，位于文档编辑器顶部），点击可重新触发全文翻译，不影响文档展示。

### 5.7.5 与【我的翻译库】联动规则

文档翻译完成后，自动同步至【我的翻译库】「文档」Tab，同步后显示文档名称、翻译时间，点击该文档可直接跳转至【文档翻译结果页】，查看原文 / 译文，不支持文档编辑、删除（贴合 MVP 阶段约束）。

### 5.7.6 通用约束（贴合全局规范）

1. 视觉规范：所有按钮、Loading、Toast 提示、Tab 样式，均与首页、网页翻译页、文本翻译页统一，蓝色主题、胶囊圆角、轻量化动效，无冗余装饰；

2. 权限规范：仅申请存储权限（用于文档上传、导出），无多余权限申请，贴合全局权限规范；

3. 页面特性：【文档翻译结果页】为非一次性页面，返回后再次进入，可保留当前翻译状态（原文 / 译文切换状态），无需重新翻译；

4. 操作约束：不添加任何非核心操作（如分享、收藏），贴合 MVP 阶段极简定位，仅保留上传、翻译、查看、导出、复制、搜一搜核心功能。

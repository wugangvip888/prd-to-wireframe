# Priority Rules

## 目标

页面元素权重排序用于把 Intent 转为可执行的线框图层级判断。
生成时必须同时参考 PRD 和 Intent：PRD 提供业务硬约束，Intent 提供页面目标、视觉重心和主要操作判断。

## 分级

```text
P0：页面核心目标、第一视觉重心、用户进入后首先处理的对象。
P1：完成核心目标所需的主要操作、高频控制、主路径控件。
P2：辅助操作、状态反馈、次要入口、非主路径但需要保留的功能。
P3：弱提示、背景、安全区、占位、低优先级信息。
```

## 判断规则

```text
每个页面必须至少有一个 P0。
P0 不宜过多，通常为 1 到 2 个核心区域。
页面级主导航可以是 P0，但必须说明理由。
能直接推动页面目标完成的操作优先为 P1。
只帮助理解或辅助跳转的内容优先为 P2。
不会影响主任务完成的提示、背景和占位优先为 P3。
不得用 P0/P1/P2/P3 表达最终视觉风格；它只表达信息层级和操作优先级。
```

## 输出格式

```text
# {page_id} Priority Map

## 页面目标
xxx

## P0
- element_id：原因

## P1
- element_id：原因

## P2
- element_id：原因

## P3
- element_id：原因
```

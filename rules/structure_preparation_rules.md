# Structure Preparation Rules

## 目标

本文件定义线框图阶段如何为后续结构化设计做准备。

它只约束预备结构，不做最终设计系统决策。

后续目标包括：

```text
高保真设计稿绘制
组件候选识别
Token 采样
Pattern 归纳
AI 后续生成页面
```

## 与其他规则的边界

```text
wireframe_rules.md：定义线框图 Frame、尺寸、排布、命名、层级和系统 UI 规则。
structure_preparation_rules.md：定义线框图如何为组件、Token、Pattern 做结构准备。
naming_rules.md（后续阶段）：定义最终页面实例、复用组件和 Pattern 命名体系。
design_rules.md（后续阶段）：定义高保真设计稿绘制、Auto Layout、状态和设计值收敛规则。
```

本项目只执行前两类规则，不在当前阶段落地最终命名体系和高保真设计规范。

## 组件候选预备规则

以下结构应保持为独立 Frame，以便后续判断是否抽取为组件：

```text
跨页面重复出现的控件或模块。
页面内重复出现的列表项、卡片、Tab、按钮、输入区、筛选区。
有明确状态变化的控件。
有“背景 + 图标/文字”组合的稳定结构。
后续可能作为独立交互单元复用的局部结构。
```

执行要求：

```text
线框图阶段可以使用 component_candidate 作为备注或记录字段，但不得直接确定最终组件名。
视觉相似不等于必须合并为同一组件。
业务语义、内容结构、交互行为、状态差异都可能导致组件拆分。
边界模糊项必须进入 run record 或人工确认点，不得由 AI 在线框图阶段最终决定。
```

禁止：

```text
不在线框图阶段建立最终组件库。
不因为两个结构长得像就强行合并。
不把一次性页面局部结构直接命名为复用组件。
不使用“组件-xxx”作为正式线框图图层名。
```

## Token 采样准备规则

线框图不生成最终 Token，但必须让后续采样有清晰结构来源。

执行要求：

```text
可采样结构必须有稳定父级 Frame。
尺寸、间距、圆角、描边、灰阶等线框图表达应挂在合理父级或明确元素上。
按钮、输入框、胶囊、卡片、Tab、列表项等结构必须有可追溯父级。
同类控件在同一状态和同一优先级下应保持尺寸、灰阶、描边和间距一致。
不同尺寸或灰阶必须能从 P0/P1/P2/P3、状态或业务场景解释。
```

图标和插图准备：

```text
图标只采父级容器结构，不采内部小几何。
图标按钮必须保留 icon_container，作为后续尺寸、圆角、灰阶和间距判断来源。
图标内部说明文字只用于线框图功能表达，不作为最终图标资产或文字 Token。
```

禁止：

```text
不把线框图灰阶当成最终颜色 Token。
不把线框图临时尺寸直接当成最终尺寸 Token。
不为无法复用的一次性内容块强行生成 Token 语义。
```

## Pattern 候选准备规则

以下结构应保留清晰模块边界，便于后续归纳 Pattern：

```text
输入任务流。
搜索 / 筛选 / 分类结构。
列表 + 详情结构。
上传 / 加载 / 结果展示流程。
源内容 / 目标内容对照结构。
工具栏 / 操作栏 / 底部导航结构。
多状态页面的状态切换结构。
```

执行要求：

```text
跨页面重复的信息结构应尽量保持同一层级深度。
跨页面重复的模块命名应尽量稳定。
同一 Pattern 候选中的 Region 和 Element 应可从 Frame 层级中识别。
如果某个 Pattern 候选只在一个页面出现，可先保留页面结构，不强行上升为 Pattern。
```

禁止：

```text
不在线框图阶段生成最终 Pattern 名。
不把单个页面的一次性布局强行沉淀为 Pattern。
不因为布局相似就忽略业务语义差异。
```

## 状态与变体准备规则

```text
页面级状态必须拆成独立 Page Frame。
同一页面多状态必须相邻排列。
组件级状态不得混入页面 Frame；如需表达，放入单独组件状态展示区或后续阶段处理。
关键状态必须语义化命名，不使用位置词或数字尾缀。
只为真实设计稿、命名系统、组件、Token 或 Pattern 需要的状态建 Frame。
```

当前阶段不做：

```text
不为流程完整性强补状态。
不为业务可能性强补状态。
不生成最终 Variant 结构。
不决定最终状态命名体系。
```

## 记录要求

如果线框图生成时发现以下情况，必须写入 run record 或执行摘要：

```text
组件候选边界不明确。
疑似 Pattern 但证据不足。
同类结构是否合并存在不确定性。
某个尺寸、灰阶、间距差异无法从 P0/P1/P2/P3 或状态解释。
PRD 要求的系统 UI / 安全区表达可能影响后续结构采样。
```

## 自检清单

生成线框图后，必须检查：

```text
组件候选是否有独立 Frame。
Token 采样所需父级 Frame 是否清晰。
重复结构的层级和命名是否一致。
Pattern 候选是否保留了模块边界。
页面级状态是否独立 Frame。
组件级状态是否没有混入页面 Frame。
是否误把线框图灰阶、临时尺寸或占位结构当作最终设计系统结论。
```
